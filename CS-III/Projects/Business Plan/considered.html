Proposals Considered

Aiden Valva's Proposal

I considered this proposal because of its obvious ability to generate revenue. In the age of big data, the ability to analyze social media to estimate consumer interest is extremely valuable. As such, many businesses would be willing to pay large sums of money for this software to maximize profits.

I decided to reject this proposal because I created a Twitter bot for last year’s ISP. As such, I won’t learn as much from making another social media bot.

Akshay Vemulapalli's Proposal

I considered this proposal because the project uses AI to create a useful functionality: keeping users awake. I’d like to incorporate AI into my ISP this year because of my interest in the field and because I understand the theory behind AI, but have relatively little experience in implementing machine learning algorithms.

I decided to reject this proposal because I don’t think many people would actually use this product. The stigma around digital surveillance is on the rise, so many potential users could be turned off by facial recognition software. Also, simply repeatedly waking someone up is unlikely to keep them awake, and most people would find more success by simply drinking coffee or energy drinks.

Andrew Liu's Proposal

I considered this proposal because I like roguelike games and there exists an obvious path to generating revenue. With the recent popularity of roguelike games, there exists a clear consumer base for a game like this. Also, there are people willing to pay, as evidenced by paid roguelike games on steam.

I decided to reject this proposal because I’m not interested in game development. Although it would be interesting to make a game, many of the skills I’d learn wouldn’t be applicable to my future career.

Berkley Voss' Proposal

I considered this proposal because the game concept seems unique. Also, for the same reasons as the last proposal, a game such as this clearly has the potential to be profitable.

I decided to reject this proposal because the actual coding process for this game wouldn’t be particularly challenging. Most of the difficulty in creating this product would revolve around creative writing or game design, both of which fall outside my goals for my ISP.

Brett Kaplan's Proposal

I considered this proposal because I think it has the potential to massively streamline the process of getting food at the cafeteria. Recently, many students have been cutting in line in the cafeteria. Along with the already slow lines, this makes it nearly impossible to get food within the lunch period. Online ordering would mostly solve this problem.

I decided to reject this proposal because I doubt Allen ISD will be open-minded enough to accept a proposal such as this. Public school systems tend to stick to the norm, so an online ordering system such as this likely wouldn’t be implemented anytime soon.

Cameron Farley's Proposal

I considered this proposal because it has a clear practical application and path to revenue. Teachers spend precious minutes every class period taking attendance, so a product such as this would save hours of teaching time over the course of a year for both teachers and students. Additionally, using software to take attendance would simplify the currently archaic tardy system.

I decided to reject this proposal because, again, it is unlikely that Allen ISD would accept such a proposal for the reasons listed above. Also, some students may dislike the change because it will decrease the time available at the beginning of classes to prepare for class.

Devin Wingfield's Proposal

I considered this proposal because it’s useful and has a clear path to profitability. Firstly, many people are interested in learning how to code, so there is a large potential user base. Also, the Duolingo model has proven the profitability of such a service.

I decided to reject this proposal because the process of learning to code isn’t totally analogous to learning a new language. Therefore, using an app to teach coding languages would probably be ineffective. For example, learning the syntax of a language would necessitate typing, but typing on a phone is cumbersome.

Evan Ye's Proposal

I considered this proposal because the end product would be extremely useful. Also, many programmers and companies would pay thousands of dollars for a program that can translate code from one language to another, considering programmers are already paid handsomely for this work.

I decided to reject this proposal because it’s nearly impossible to create. Ignoring the obvious case where one language is built on another, flawlessly translating code from one language to another would require groundbreaking developments in AI. Many companies (Facebook, for example) have already tried to create such a product, but none have succeeded in creating a perfect code translator.

Matthew Roman's Proposal

I considered this proposal because a platform that connects CS students and teachers would be useful. Also, advertising could be used to generate revenue given a sufficient user base.

I decided to reject this proposal because social media already have options to create smaller communities. Reddit and Facebook are examples of social media that already provide opportunities for CS students and teachers to interact. Therefore, it’s unlikely that this product would catch on.

Gautam Narayan's Proposal

I considered this proposal because it has the potential to generate revenue through stock trading and subscriptions. Also, it uses data from multiple sources to inform predictions on stocks, which would help me learn how to combine lots of information using computers.

I decided to accept this proposal because it uses AI, which I want to learn more about. Also, the numeric data of the stock market allows the AI to train on objective data to inform its sentiment analysis. Because this project ties multiple neural network architectures (including DNNs and RNNs) into one goal, I’d learn about the implementation of a wide variety of machine learning techniques.
