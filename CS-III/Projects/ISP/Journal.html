2-Jan-2023

(Software) Development Engineer

Software engineers design and maintain software. This requires technical skills (e.g. knowledge of programming languages and data structures) and problem-solving creativity. They are the most basic unit of a software development team because they create software at the beginning of the process from idea to product. Software engineers tend to specialize in one or more areas, such as a programming language or operating system.

Of the roles listed, I'm most familiar with the software engineer role. I've been programming for three years now, so I have a good understanding of a variety of useful languages, including Python, Swift, Java, JavaScript, and C++. Also, because of my experience in competitive programming, I understand the benefits and drawbacks of various algorithms and data structures. Also, I'll be able to apply my general problem-solving skills to coding.

As a software engineer, one challenge I'll have to deal with is coding within the confines of the project. I tend to work on projects independently, so I'll need to get better at working with others on software. I can do this by communicating clearly with my team members and asking for clarification when I don't understand something. In coding, my biggest difficulty will be with using APIs. Because we plan to scrape text from reddit.com, I'll need to learn how to use the Reddit API. I can improve in this area simply by reading documentation.

QA Engineer

QA engineers ensure that software meets quality standards before being released to the public. This involves actively looking for bugs and oversights. QA engineers must deeply understand how the software works to see why it has certain issues. QA engineers work closely with software engineers to document and test the software along the software development process. QA engineers may also write code to automate software testing.

I'm good at spotting mistakes in code, but as a QA engineer, I'll need to be able to use our software like an end user would and find bugs that way, which will be more difficult for me. Also, when I find problems with code, I generally know how to fix them, but I don't have much experience debugging other people's code. Additionally, I don't enjoy playing around with the end product more than I have to.

One challenge I'll encounter as a QA engineer is communicating with my team members effectively. As a QA engineer, when I find bugs, I'll need to explain the specific steps I took to find the bug so that the engineers can reproduce my actions. Also, as a QA engineer, I'll have to ask engineers to fix their own code more than I fix their code for them, so I'll need to be specific about the bug in the code and what the fixed version should act like. To do this, I'll specify what the code is doing wrong and what it should be doing.

Project Manager

Project managers lead the software development team. This involves communicating with shareholders and defining goals within time and budget restraints. Software engineers must be able to manage money effectively, correctly evaluate team members' performance, study their industry to keep up to date with new strategies, and organize team members from various departments.

I'm good at dividing work among people and ensuring everyone is doing their part, so I'd be able to organize the team as a project manager well. However, I don't have much experience communicating with the consumer very much, so I'd need to learn how to please shareholders. Also, I'd need to research developments in the field I'm working in so that our team maintains a competitive advantage. Networking and thorough research would make this easier.

As a project manager, one challenge I'll encounter is assessing if any team members aren't working effectively. To solve this problem, I'll focus on my team members' results—that is, I'll focus on their code to determine if they are contributing to the team. If anyone is producing subpar work in terms of quality or quantity, I'll explain the problem to them so that they can improve. Another challenge I'll encounter is communicating with shareholders. To overcome this, I'll maintain a flexible schedule to allow for any changes in my plans due to shareholders' requests.

Product Manager

Like project managers, product managers also have a broad set of responsibilities. Product managers study consumers and evaluate how the product will serve their needs. They must also set guidelines for what a final product should look like, in accordance with the consumer market. Like the project manager, the product manager must also work with shareholders to describe their view of the ideal product.

I'm good at recognizing a product's utility, so I'd be good at deciding what to do to improve our product. Also, because of my knowledge of software design, I'd be able to make an effective plan for our team to create the final product. However, because I don't have experience working with shareholders, I'll need to consider their wishes in my vision for the end product.

As a product manager, one challenge I'll encounter is determining when a software engineer isn't creating software that matches the goal for the final product. To solve this problem, I'll describe my goal for the final product very clearly, so when an engineer fails to produce what I've asked for, I can reference my original description to explain why their software can't be utilized to create the final product.
